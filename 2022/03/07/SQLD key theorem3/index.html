SQLD key theorem3

SQLD н•өмӢ¬м •лҰ¬3

<мҲңмҲҳ кҙҖкі„м—°мӮ°мһҗмҷҖ SQL л¬ёмһҘ л№„көҗ>

  • SELECT м—°мӮ°мқҖ WHERE м ҲлЎң кө¬нҳ„
  • PROJECT м—°мӮ°мқҖ SELECT м ҲлЎң кө¬нҳ„
  • (NATURAL) JOIN м—°мӮ°мқҖ лӢӨм–‘н•ң JOIN кё°лҠҘмңјлЎң кө¬нҳ„
  • DIVIDE м—°мӮ°мқҖ нҳ„мһ¬ мӮ¬мҡ©лҗҳм§Җ м•ҠмқҢ.

<ANSI/ISO-SQLм—җм„ң н‘ңмӢңн•ҳлҠ” FROM м Ҳмқҳ JOINнҳ•нғң>

  • INNER JOIN
  • NATURAL JOIN
  • USING мЎ°кұҙм Ҳ
  • ON мЎ°кұҙм Ҳ
  • CROSS JOIN
  • OUTER JOIN(LEFT,RIGHT, FULL)

в–¶ INNER JOIN

INNER JOINмқҖ OUTER(мҷёл¶Җ) JOINкіј лҢҖл№„н•ҳм—¬ лӮҙл¶Җ JOINмқҙлқјкі  н•ҳл©° JOIN мЎ°кұҙм—җм„ң лҸҷмқјн•ң к°’мқҙ мһҲлҠ” н–үл§Ң л°ҳнҷҳн•ңлӢӨ.

в–¶ CROSS JOIN

н…Ңмқҙлё” к°„ JOIN мЎ°кұҙмқҙ м—ҶлҠ” кІҪмҡ° мғқкёё мҲҳ мһҲлҠ” лӘЁл“  лҚ°мқҙн„°мқҳ мЎ°н•©мқ„ л§җн•ңлӢӨ. кІ°кіјлҠ” м–‘мӘҪ м§‘н•©мқҳ M*N кұҙмқҳ лҚ°мқҙн„° мЎ°н•©мқҙ л°ңмғқн•ңлӢӨ.

в–¶ LEFT OUTER JOIN

мЎ°мқё мҲҳн–үмӢң лЁјм Җ н‘ңкё°лҗң мўҢмёЎ н…Ңмқҙлё”м—җ н•ҙлӢ№н•ҳлҠ” лҚ°мқҙн„°лҘј лЁјм Җ мқҪмқҖ нӣ„, лӮҳмӨ‘ н‘ңкё°лҗң мҡ°мёЎ н…Ңмқҙлё”м—җм„ң JOIN лҢҖмғҒ лҚ°мқҙн„°лҘј мқҪм–ҙ мҳЁлӢӨ. мҰү, Table AмҷҖ Bк°Җ мһҲмқ„ л•Ң(Table вҖҳAвҖҷк°Җ кё°мӨҖмқҙ лҗЁ), AмҷҖ BлҘј л№„көҗн•ҙм„ң Bмқҳ JOIN м№јлҹјм—җм„ң к°ҷмқҖ к°’мқҙ мһҲмқ„ л•Ң к·ё н•ҙлӢ№ лҚ°мқҙн„°лҘј к°Җм ёмҳӨкі , Bмқҳ JOIN м№јлҹјм—җм„ң к°ҷмқҖ к°’мқҙ м—ҶлҠ” кІҪмҡ°м—җлҠ” B н…Ңмқҙлё”м—җм„ң к°Җм ёмҳӨлҠ” м№јлҹјл“ӨмқҖ NULL к°’мңјлЎң мұ„мҡҙлӢӨ.

в–¶ FULL OUTER JOIN

мЎ°мқё мҲҳн–үмӢң мўҢмёЎ, мҡ°мёЎ н…Ңмқҙлё”мқҳ лӘЁл“  лҚ°мқҙн„°лҘј мқҪм–ҙ JOINн•ҳм—¬ кІ°кіјлҘј мғқм„ұн•ңлӢӨ. мҰү, TABLE AмҷҖ Bк°Җ мһҲмқ„ л•Ң(TABLE вҖҳAвҖҷ, вҖҳBвҖҷ лӘЁл‘җ кё°мӨҖмқҙ лҗЁ.), RIGHT OUTER JOINкіј LEFT OUTER JOINкіј LEFT OUTER JOINмқҳ кІ°кіјлҘј н•©м§‘н•©мңјлЎң мІҳлҰ¬н•ң кІ°кіјмҷҖ лҸҷмқјн•ҳлӢӨ.

<OUTER JOIN л¬ёмһҘ мҳҲмӢң>

  • LEFT OUTER JOIN

    SELECT X.KEY1, Y.KEY2

    FROM TAB1 X LEFT OUTER JOIN TAB2 Y

    ON (X.KEY1=Y.KEY2)

  • RIGHT OUTER JOIN

    SELECT X.KEY1, Y.KEY2

    FROM TAB1 X RIGHT OUTER JOIN TAB2 Y

    ON(X.KEY1=Y.KEY2)

  • FULL OUTER JOIN

    SELECT X.KEY1, Y.KEY2

    FROM TAB1 X FULL OUTER JOIN TAB2 Y

    ON (X.KEY1=Y.KEY2)0

UNION ALL м—¬лҹ¬ к°ңмқҳ SQLл¬ёмқҳ кІ°кіјм—җ лҢҖн•ң н•©м§‘н•©мңјлЎң мӨ‘ліөлҗң н–үлҸ„ к·ёлһҳлҸ„ кІ°кіјлЎң н‘ңмӢңлҗңлӢӨ. мҰү, лӢЁмҲңн•ң кІ°кіјл§Ң н•©міҗлҶ“мқҖ кІғмқҙлӢӨ. к°ңлі„ SQLл¬ёмқҳ кІ°кіјк°Җ м„ңлЎң мӨ‘ліөлҗҳм§Җ м•ҠлҠ” кІҪмҡ°, UNIONкіј кІ°кіјк°Җ лҸҷмқјн•ҳлӢӨ.
INTERSECT м–ҙлҹ¬ к°ңмқҳ SQLл¬ёмқҳ кІ°кіјм—җ лҢҖн•ң көҗм§‘н•©мқҙлӢӨ. мӨ‘ліөлҗң н–үмқҖ н•ҳлӮҳмқҳ н–үмңјлЎң л§Ңл“ лӢӨ.

в–¶ PRIOR

CONNECT BY м Ҳм—җм„ң мӮ¬мҡ©лҗҳл©°, нҳ„мһ¬ мқҪлҠ” м№јлҹјмқ„ м§Җм •н•ңлӢӨ. PRIOR мһҗмӢқ = л¶ҖлӘЁ нҳ•нғңлҘј мӮ¬мҡ©н•ҳл©ҙ кі„мёөкө¬мЎ°м—җм„ң л¶ҖлӘЁ лҚ°мқҙн„°м—җм„ң мһҗмӢқ лҚ°мқҙн„°(л¶ҖлӘЁ вҶ’ мһҗмӢқ) л°©н–ҘмңјлЎң м „к°ңн•ҳлҠ” мҲңл°©н–Ҙ м „к°ңлҘј н•ңлӢӨ. к·ёлҰ¬кі  PRIOR л¶ҖлӘЁ = мһҗмӢқ нҳ•нғңлҘј мӮ¬мҡ©н•ҳл©ҙ л°ҳлҢҖлЎң мһҗмӢқ лҚ°мқҙн„°м—җм„ң л¶ҖлӘЁ лҚ°мқҙн„°(мһҗмӢқ вҶ’л¶ҖлӘЁ) л°©н–ҘмңјлЎң м „к°ңн•ҳлҠ” м—ӯл°©н–Ҙ м „к°ңлҘј н•ңлӢӨ.

в–¶ кі„мёөнҳ• м§Ҳмқҳ : н…Ңмқҙлё”м—җ кі„мёөнҳ• лҚ°мқҙн„°к°Җ мЎ°мһ¬н•ҳлҠ” кІҪмҡ° лҚ°мқҙн„°лҘј мЎ°нҡҢн•ҳкё° мң„н•ҙм„ң мӮ¬мҡ©.

в–¶ м…Җн”„ мЎ°мқё(Self Join)

лҸҷмқј н…Ңмқҙлё” мӮ¬мқҙмқҳ мЎ°мқёмқ„ л§җн•ңлӢӨ. л”°лқјм„ң FROMм Ҳм—җ лҸҷмқј н…Ңмқҙлё”мқҙ 2лІҲ мқҙмғҒ лӮҳнғҖлӮңлӢӨ. лҸҷмқј н…Ңмқҙлё” мӮ¬мқҙмқҳ мЎ°мқёмқ„ мҲҳн–үн•ҳл©ҙ н…Ңмқҙлё”кіј м№јлҹј мқҙлҰ„мқҙ лӘЁл‘җ лҸҷмқјн•ҳкё° л•Ңл¬ём—җ мӢқлі„мқ„ мң„н•ҙ л°ҳл“ңмӢң н…Ңмқҙлё” лі„м№ӯ(Alias)лҘј мӮ¬мҡ©н•ҙм•ј н•ңлӢӨ.

м„ңлёҢмҝјлҰ¬ мў…лҘҳ м„ӨлӘ…
Sing Row м„ңлёҢмҝјлҰ¬(лӢЁмқјн–ү м„ңлёҢмҝјлҰ¬) м„ңлёҢмҝјлҰ¬мқҳ мӢӨн–ү кІ°кіјк°Җ н•ӯмғҒ 1кұҙ мқҙн•ҳмқё м„ңлёҢмҝјлҰ¬лҘј мқҳлҜён•ңлӢӨ. лӢЁмқј н–ү м„ңлёҢмҝјлҰ¬лҠ” лӢЁмқј н–ү л№„көҗ м—°мӮ°мһҗмҷҖ н•Ёк»ҳ мӮ¬мҡ©лҗңлӢӨ. лӢЁмқј н–ү л№„көҗм—°мӮ°мһҗм—җлҠ” =,<,< =,>,> =, <>мқҙ мһҲлӢӨ.
Multi Row м„ңлёҢмҝјлҰ¬(лӢӨмӨ‘ н–ү м„ңлёҢмҝјлҰ¬) м„ңлёҢмҝјлҰ¬мқҳ мӢӨн–ү кІ°кіјк°Җ м—¬лҹ¬ кұҙмқё м„ңлёҢмҝјлҰ¬лҘј мқҳлҜён•ңлӢӨ. лӢӨмӨ‘ н–ү м„ңлёҢмҝјлҰ¬лҠ” лӢӨмӨ‘ н–ү л№„көҗ м—°мӮ°мһҗмҷҖ н•Ёк»ҳ мӮ¬мҡ©лҗңлӢӨ. лӢӨмӨ‘ н–ү л№„көҗм—°мӮ°мһҗм—җлҠ” IN, ALL, ANY, SOME, EXISTSк°Җ мһҲлӢӨ.
Multi Column м„ңлёҢмҝјлҰ¬(лӢӨмӨ‘ м№јлҹј м„ңлёҢмҝјлҰ¬) м„ңлёҢмҝјлҰ¬мқҳ мӢӨн–ү кІ°кіјлЎң м—¬лҹ¬ м№јлҹјмқ„ л°ҳнҷҳн•ңлӢӨ. л©”мқёмҝјлҰ¬мқҳ мЎ°кұҙм Ҳм—җ м—¬лҹ¬ м№јлҹјмқ„ лҸҷмӢңм—җ л№„көҗн•  мҲҳ мһҲлӢӨ. м„ңлёҢмҝјлҰ¬мҷҖ л©”мқёмҝјлҰ¬м—җм„ң л№„көҗн•ҳкі мһҗ н•ҳлҠ” м№јлҹј к°ңмҲҳмҷҖ м№јлҹјмқҳ мң„м№ҳк°Җ лҸҷмқјн•ҙм•ј н•ңлӢӨ.

в–¶ мқёлқјмқё л·°(InIine View)

FROM м Ҳм—җм„ң мӮ¬мҡ©н•ҳлҠ” м„ңлёҢмҝјлҰ¬лҘј л§җн•ңлӢӨ. м„ңлёҢмҝјлҰ¬мқҳ кІ°кіјк°Җ л§Ҳм№ҳ мӢӨн–ү мӢңм—җ лҸҷм ҒмңјлЎң мғқм„ұлҗң н…Ңмқҙлё”мқё кІғмІҳлҹј мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ. мқёлқјмқё л·°лҠ” SQLл¬ёмқҙ мӢӨн–ү лҗ  л•Ңл§Ң мһ„мӢңм ҒмңјлЎң мғқм„ұлҗҳлҠ” лҸҷм Ғмқё л·°мқҙкё° л•Ңл¬ём—җ лҚ°мқҙн„°лІ мқҙмҠӨм—җ н•ҙлӢ№ м •ліҙк°Җ м ҖмһҘлҗҳм§Җ м•ҠлҠ”лӢӨ.

м„ңлёҢмҝјлҰ¬мқҳ кІ°кіјк°Җ л§Ҳм№ҳ мӢӨн–ү мӢңм—җ лҸҷм ҒмңјлЎң мғқм„ұлҗң н…Ңмқҙлё”мқё кІғмІҳлҹј мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ. мқёлқјмқё л·°лҠ” SQLл¬ёмқҙ мӢӨн–үлҗ  л•Ңл§Ң мһ„мӢңм ҒмңјлЎң мғқм„ұлҗҳлҠ” лҸҷм Ғмқё л·°мқҙкё° л•Ңл¬ём—җ лҚ°мқҙн„°лІ мқҙмҠӨм—җ н•ҙлӢ№ м •ліҙк°Җ м ҖмһҘлҗҳм§Җ м•ҠлҠ”лӢӨ.

<л·° мӮ¬мҡ©мқҳ мһҘм җ>

  • лҸ…лҰҪм„ұ : н…Ңмқҙлё” кө¬мЎ°к°Җ ліҖкІҪлҗҳм–ҙлҸ„ л·°лҘј мӮ¬мҡ©н•ҳлҠ” мқ‘мҡ© н”„лЎңк·ёлһЁмқҖ ліҖкІҪн•ҳм§Җ м•Ҡм•„лҸ„ лҗңлӢӨ.
  • нҺёлҰ¬м„ұ : ліөмһЎн•ң м§ҲмқҳлҘј л·°лЎң мғқм„ұн•ЁмңјлЎңмҚЁ кҙҖл Ё м§ҲмқҳлҘј лӢЁмҲңн•ҳкІҢ мһ‘м„ұн•  мҲҳ мһҲлӢӨ. лҳҗн•ң н•ҙлӢ№ нҳ•нғңмқҳ SQLл¬ёмқ„ мһҗмЈј мӮ¬мҡ©н•  л•Ң л·°лҘј мқҙмҡ©н•ҳл©ҙ нҺёлҰ¬н•ҳкІҢ мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.
  • ліҙм•Ҳм„ұ : м§Ғмӣҗмқҳ кёүм—¬м •ліҙмҷҖ к°ҷмқҙ мҲЁкё°кі  мӢ¶мқҖ м •ліҙк°Җ мЎҙмһ¬н•ңлӢӨл©ҙ, л·°лҘј мғқм„ұн•  л•Ң н•ҙлӢ№ м№јлҹјмқ„ л№јкі  мғқм„ұн•ЁмңјлЎңмҚЁ мӮ¬мҡ©мһҗм—җкІҢ м •ліҙлҘј к°җм¶ң мҲҳ мһҲлӢӨ.

в–¶ Grouping Columns

к°Җм§Ҳ мҲҳ мһҲлҠ” лӘЁл“  кІҪмҡ°м—җ лҢҖн•ҳм—¬ Subtotalмқ„ мғқм„ұн•ҙм•ј н•ҳлҠ” кІҪмҡ°м—җлҠ” CUBEлҘј мӮ¬мҡ©н•ҳлҠ” кІғмқҙ л°”лһҢм§Ғн•ҳлӮҳ, ROLLUPм—җ л№„н•ҙ мӢңмҠӨн…ңм—җ л§ҺмқҖ л¶ҖлӢҙмқ„ мЈјлҜҖлЎң мӮ¬мҡ©н•ҙ мЈјмқҳ н•  кІғ

в–¶ CUBE

кІ°н•©к°ҖлҠҘн•ң лӘЁл“  к°’м—җ лҢҖн•ҳм—¬ лӢӨм°Ёмӣҗ м§‘кі„лҘј мғқм„ұн•ңлӢӨ. CUBEлҸ„ кІ°кіј м—җ лҢҖн•ң м •л ¬мқҙ н•„мҡ”н•ң кІҪмҡ°лҠ” ORDER BY м Ҳм—җ лӘ…мӢңм ҒмңјлЎң м •л ¬ м№јлҹјмқҙ н‘ңмӢң лҗҳм–ҙм•ј н•ңлӢӨ.

в–¶ GROUPING SETS

лӢӨм–‘н•ң мҶҢкі„ м§‘н•©мқ„ л§Ңл“Ө мҲҳ мһҲлҠ”лҚ°, GROUPING SETSм—җ н‘ңмӢңлҗң мқёмҲҳл“Өм—җ лҢҖн•ң к°ңлі„ м§‘кі„лҘј кө¬н•  мҲҳ мһҲмңјл©°, мқҙл•Ң н‘ңмӢңлҗң мқёмҲҳл“Ө к°„м—җлҠ” кі„мёө кө¬мЎ°мқё ROLLUPкіјлҠ” лӢ¬лҰ¬ нҸүл“ұн•ң кҙҖкі„мқҙлҜҖлЎң мқёмҲҳмқҳ мҲңм„ңк°Җ л°”лҖҢм–ҙлҸ„ кІ°кіјлҠ” к°ҷлӢӨ.

к·ёлҰ¬кі  GROUPING SETS н•ЁмҲҳлҸ„ кІ°кіјм—җ лҢҖн•ң м •л ¬мқҙ н•„мҡ”н•ң кІҪмҡ°лҠ” ORDER BY м Ҳм—җ лӘ…мӢңм ҒмңјлЎң м •л ¬ м№јлҹјмқҙ н‘ңмӢң лҗҳм–ҙм•ј н•ңлӢӨ.

в–¶ RANK н•ЁмҲҳ

ORDER BYлҘј нҸ¬н•Ён•ң QUERY л¬ём—җм„ң нҠ№м • м№јлҹјм—җ лҢҖн•ҙ мҲңмң„лҘј кө¬н•ҳлҠ” н•ЁмҲҳмқҙл©° лҸҷмқён•ң к°’м—җ лҢҖн•ҙм„ңлҠ” лҸҷмқјн•ң мҲңмң„лҘј л¶Җм—¬н•ҳкІҢ лҗңлӢӨ.

в–¶DENSE RANK н•ЁмҲҳ

RANK н•ЁмҲҳмҷҖ нқЎмӮ¬н•ҳлӮҳ, лҸҷмқјн•ң мҲңмң„лҘј н•ҳлӮҳмқҳ кұҙмҲҳлЎң м·Ёкёүн•ҳлҠ” кІғмқҙ лӢӨлҘё м җмқҙлӢӨ.

в–¶ ROW_NUMBER н•ЁмҲҳ

RANKлӮҳ DENSE_RANK н•ЁмҲҳк°Җ лҸҷмқјн•ң к°’м—җ лҢҖн•ҙм„ңлҠ” лҸҷмқјн•ң мҲңмң„лҘј л¶Җм—¬н•ҳлҠ”лҚ° л°ҳн•ҙ, лҸҷмқјн•ң к°’мқҙлқјлҸ„ кі мң н•ң мҲңмң„лҘј ліҙмң н•ңлӢӨ.

<PL/SQLмқҳ нҠ№м§•>

  • PL/SQLмқҖ Block кө¬мЎ°лЎң лҗҳм–ҙмһҲм–ҙ к°Ғ кё°лҠҘлі„лЎң лӘЁл“Ҳнҷ”к°Җ к°ҖлҠҘн•ҳлӢӨ.
  • ліҖмҲҳ, мғҒмҲҳ л“ұмқ„ м„ м–ён•ҳм—¬ SQL л¬ёмһҘ к°„ к°’мқ„ көҗнҷҳн•ңлӢӨ.
  • IF, LOOP л“ұмқҳ м Ҳм°Ёнҳ• м–ём–ҙлҘј мӮ¬мҡ©н•ҳм—¬ м Ҳм°Ём Ғмқё н”„лЎңк·ёлһЁмқҙ к°ҖлҠҘн•ҳлҸ„лЎқ н•ңлӢӨ.
  • DBMS м •мқҳ м—җлҹ¬лӮҳ мӮ¬мҡ©мһҗ м •мқҳ м—җлҹ¬лҘј м •мқҳн•ҳм—¬ мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.
  • PL/SQLмқҖ Oracleм—җ лӮҙмһҘлҗҳм–ҙ мһҲмңјлҜҖлЎң Oracleкіј PL/SQLмқ„ м§Җмӣҗн•ҳлҠ” м–ҙл–Ө м„ңлІ„лЎңлҸ„ н”„лЎңк·ёлһЁмқ„ мҳҜкёё мҲҳ мһҲлӢӨ.
  • PL/SQLмқҖ мқ‘мҡ© н”„лЎңк·ёлһЁмқҳ м„ұлҠҘмқ„ н–ҘмғҒмӢңнӮЁлӢӨ.
  • PL/SQLмқҖ м—¬лҹ¬ SQLл¬ёмһҘмқ„ BlockмңјлЎң л¬¶кі  н•ң лІҲм—җ Block м „л¶ҖлҘј м„ңлІ„лЎң ліҙлӮҙкё° л•Ңл¬ём—җ нҶөмӢ лҹүмқ„ мӨ„мқј мҲҳ мһҲлӢӨ.

в–¶ м ҖмһҘ лӘЁл“Ҳ(Stored Module)

SQL л¬ёмһҘмқ„ лҚ°мқҙн„°лІ мқҙмҠӨ м„ңлІ„м—җ м ҖмһҘн•ҳм—¬ мӮ¬мҡ©мһҗмҷҖ м• н”ҢлҰ¬мјҖмқҙм…ҳ мӮ¬мқҙм—җм„ң кіөмң н•  мҲҳ мһҲлҸ„лЎқ л§Ңл“  мқјмў…мқҳ SQL м»ҙнҸ¬л„ҢнҠё н”„лЎңк·ёлһЁмқҙл©°, лҸ…лҰҪм ҒмңјлЎң мӢӨн–үлҗҳкұ°лӮҳ лӢӨлҘё н”„лЎңк·ёлһЁмңјлЎңл¶Җн„° мӢӨн–үлҗ  мҲҳ мһҲлҠ” мҷ„м „н•ң мӢӨн–ү н”„лЎңк·ёлһЁмқҙлӢӨ.

Oracleмқҳ м ҖмһҘ лӘЁл“Ҳм—җлҠ” Procedure, User Defined Function, Triggerк°Җ мһҲлӢӨ.

<н”„лЎңмӢңм ҖмҷҖ нҠёлҰ¬кұ°мқҳ м°Ёмқҙм җ>

н”„лЎңмӢңм Җ нҠёлҰ¬кұ°
CREATE Procedure л¬ёлІ•мӮ¬мҡ© CREATE Trigger л¬ёлІ•мӮ¬мҡ©
EXECUTE лӘ…л №м–ҙлЎң мӢӨн–ү мғқм„ұ нӣ„ мһҗлҸҷмңјлЎң мӢӨн–ү
COMMIT, ROLLBACK мӢӨн–үк°ҖлҠҘ COMMIT, ROLLBACK мӢӨн–үм•ҲлҗЁ

в–¶ Sort Merge Join

мЎ°мқё м№јлҹјмқ„ кё°мӨҖмңјлЎң лҚ°мқҙн„°лҘј м •л ¬н•ҳм—¬ мЎ°мқёмқ„ мҲҳн–үн•ңлӢӨ. NL JoinмқҖ мЈјлЎң лһңлҚӨ м—‘м„ёмҠӨ л°©мӢқмңјлЎң лҚ°мқҙн„°лҘј мқҪлҠ” л°ҳл©ҙ Sort Merge JoinмқҖ мЈјлЎң мҠӨмә” л°©мӢқмңјлЎң лҚ°мқҙн„°лҘј мқҪлҠ”лӢӨ. Sort Merge JoinмқҖ лһңлҚӨ м—‘м„ёмҠӨлЎң NL Joinм—җм„ң л¶ҖлӢҙмқҙ лҗҳлҚҳ л„“мқҖ лІ”мң„мқҳ лҚ°мқҙн„°лҘј мІҳлҰ¬н•  л•Ң мқҙмҡ©лҗҳлҚҳ мЎ°мқёкё°лІ•мқҙлӢӨ. к·ёлҹ¬лӮҳ Sort Merge JoinмқҖ м •л ¬н•  лҚ°мқҙн„°к°Җ л§Һм•„ л©”лӘЁлҰ¬м—җм„ң лӘЁл“  м •л ¬ мһ‘м—…мқ„ мҲҳн–үн•ҳкё° м–ҙл Өмҡҙ кІҪмҡ°м—җлҠ” мһ„мӢң мҳҒм—ӯ(л””мҠӨнҒ¬)мқ„ мӮ¬мҡ©н•ҳкё° л•Ңл¬ём—җ м„ұлҠҘмқҙ л–Ём–ҙм§Ҳ мҲҳ мһҲлӢӨ.

в–¶Hash Join

мЎ°мқё м№јлҹјмқҳ мқёлҚұмҠӨлҘј мӮ¬мҡ©н•ҳм§Җ м•Ҡкё° л•Ңл¬ём—җ мЎ°мқё м№јлҹјмқҳ мқёлҚұмҠӨк°Җ мЎҙмһ¬н•ҳм§Җ м•Ҡмқ„ кІҪмҡ°м—җлҸ„ мӮ¬мҡ©н•  мҲҳ мһҲлҠ” мЎ°мқё кё°лІ•мқҙлӢӨ. Hash JoinмқҖ н•ҙмү¬ н•ЁмҲҳлҘј мқҙмҡ©н•ҳм—¬ мЎ°мқёмқ„ мҲҳн–үн•ҳкё° л•Ңл¬ём—җ вҖҳ=вҖҷлЎң мҲҳн–үн•ҳлҠ” мЎ°мқё мҰү, лҸҷл“ұ мЎ°мқём—җм„ңл§Ң мӮ¬мҡ© к°ҖлҠҘн•ҳлӢӨ.

в–¶ нҒҙлқјмқҙм–ёнҠёк°Җ м„ңлІ„ н”„лЎңм„ёмҠӨмҷҖ м—°кІ°н•ҳлҠ” Oracleмқҳ мҳҲ

  1. м „мҡ© м„ңлІ„(Dedicated Server) л°©мӢқ

  2. кіөмң  м„ңлІ„(Shared Server) л°©мӢқ

в–¶ SQL Serverм—җм„ңлҠ” м„ёк·ёлЁјнҠёлҠ” н…Ңмқҙлё”, мқёлҚұмҠӨ, UndoмІҳлҹј м ҖмһҘкіөк°„мқ„ н•„мҡ”лЎң н•ҳлҠ” лҚ°мқҙн„°лІ мқҙмҠӨ мҳӨлёҢм қнҠёлӢӨ. м ҖмһҘкіөк°„мқ„ н•„мҡ”лЎң н•ңлӢӨлҠ” кІғмқҖ н•ң к°ң мқҙмғҒмқҳ мқөмҠӨн…җнҠёлҘј мӮ¬мҡ©н•Ёмқ„ лң»н•ңлӢӨ.

в–¶ SQL Serverм—җм„ңлҠ” м„ёк·ёлЁјнҠёлқјлҠ” мҡ©м–ҙлҘј мӮ¬мҡ©н•ҳм§Җ м•Ҡм§Җл§Ң, нһҷ кө¬мЎ° лҳҗлҠ” мқёлҚұмҠӨ кө¬мЎ°мқҳ мҳӨлёҢм қнҠёк°Җ м—¬кё°м—җ мҶҚн•ңлӢӨ.

в–¶ DB лІ„нҚј мәҗмӢңм—җ к°Җн•ҙмһҗлҠ” лӘЁл“  ліҖкІҪмӮ¬н•ӯмқ„ кё°лЎқн•ҳлҠ” OracleмқҖ вҖҳRedo лЎңк·ёвҖҷлқјкі  л¶ҖлҘҙл©°, 

 SQL ServerлҠ” вҖҳ**нҠёлһҷмһӯм…ҳ лЎңк·ё**вҖҷлқјкі  л¶ҖлҘёлӢӨ.

в–¶ Response Time Analysis л°©лІ•лЎ 

Response Timeмқ„ м •мқҳн•ҳкі , CPU Timeкіј Wait Timeмқ„ к°Ғк°Ғ break down н•ҳл©ҙм„ң м„ңлІ„мқҳ мқјлҹүкіј лҢҖкё° мӢңк°„мқ„ л¶„м„қн•ҙ лӮҳк°„лӢӨ. CPU TimeмқҖ нҢҢмӢұ мһ‘м—…м—җ мҶҢл№„н•ң мӢңк°„мқём§Җ м•„лӢҲл©ҙ мҝјлҰ¬ ліём—°мқҳ мҳӨнҚјл Ҳмқҙм…ҳ мҲҳн–үмқ„ мң„н•ҙ мҶҢл№„н•ң мӢңк°„мқём§ҖлҘј л¶„м„қн•ңлӢӨ. Wait TimeмқҖ к°Ғк°Ғ л°ңмғқн•ң лҢҖкё° мқҙлІӨнҠёл“Өмқ„ л¶„м„қн•ҙ к°ҖмһҘ мӢңк°„мқ„ л§Һмқҙ мҶҢл№„н•ң мқҙлІӨнҠё мӨ‘мӢ¬мңјлЎң н•ҙкІ°л°©м•Ҳмқ„ лӘЁмғүн•ңлӢӨ.

в–¶ мҶҢн”„нҠё нҢҢмӢұкіј н•ҳл“ң нҢҢмӢұ

  • мҶҢн”„нҠё нҢҢмӢұ(Soft Parsing) : SQLкіј мӢӨн–үкі„нҡҚмқ„ мәҗмӢңм—җм„ң м°ҫм•„ кі§л°”лЎң мӢӨн–үлӢЁкі„лЎң л„ҳм–ҙк°ҖлҠ” кІҪмҡ°лҘј л§җн•Ё.
  • н•ҳл“ң нҢҢмӢұ(Hard Parsing) : SQLкіј мӢӨн–үкі„нҡҚмқ„ мәҗмӢңм—җм„ң м°ҫм§Җ лӘ»н•ҙ мөңм Ғнҷ” кіјм •мқ„ кұ°м№ҳкі  лӮҳм„ң мӢӨн–үлӢЁкі„лЎң л„ҳм–ҙк°ҖлҠ” кІҪмҡ°лҘј л§җн•Ё.

в–¶ л°”мқёл“ң ліҖмҲҳ (Bind Variable) : нҢҢлқјлҜён„° Driven л°©мӢқмңјлЎң SQLмқ„ мһ‘м„ұн•ҳлҠ” л°©лІ•мқҙ м ңкіөлҗҳлҠ”лҚ° SQLкіј мӢӨн–үкі„нҡҚмқ„ м—¬лҹ¬ к°ң мәҗмӢұн•ҳм§Җ м•Ҡкі  н•ҳлӮҳлҘј л°ҳліө мһ¬мӮ¬мҡ©н•ҳлҜҖлЎң нҢҢмӢұ мҶҢмҡ”мӢңк°„кіј л©”лӘЁлҰ¬ мӮ¬мҡ©лҹүмқ„ мӨ„м—¬мӨҖлӢӨ.

в–¶ SQL м»Өм„ңм—җ лҢҖн•ң мһ‘м—…мҡ”мІӯм—җ л”°лҘё лҚ°мқҙн„°лІ мқҙмҠӨ Callмқҳ кө¬л¶„

  1. Parse Call : SQL нҢҢмӢұмқ„ мҡ”мІӯн•ҳлҠ” Call
  2. Execute Call : SQL мӢӨн–үмқ„ мҡ”мІӯн•ҳлҠ” Call
  3. Fetch Call : SELECTл¬ёмқҳ кІ°кіј лҚ°мқҙн„° м „мҶЎмқ„ мҡ”мІӯн•ҳлҠ” Call

в–¶ мӮ¬мҡ©мһҗ м •мқҳ н•ЁмҲҳ/н”„лЎңмӢңм Җ

лӮҙмһҘ н•ЁмҲҳмІҳлҹј Native мҪ”л“ңлЎң мҷ„м „ м»ҙнҢҢмқјлҗң нҳ•нғңк°Җ м•„лӢҲм–ҙм„ң к°ҖмғҒлЁёмӢ (Virtual Machine) к°ҷмқҖ лі„лҸ„мқҳ мӢӨн–ү м—”м§„мқ„ нҶөн•ҙ мӢӨн–үлҗңлӢӨ.

мӢӨн–ү лҗ  л•Ңл§ҲлӢӨ м»Ён…ҚмҠӨнҠё мҠӨмң„м№ӯ(Context Switching)мқҙ мқјм–ҙлӮҳл©°, мқҙ л•Ңл¬ём—җ лӮҙмһҘ н•ЁмҲҳ(Built-In)лҘј нҳём¶ңн•  л•ҢмҷҖ л№„көҗн•ҙ м„ұлҠҘмқ„ мғҒлӢ№нһҲ л–Ём–ҙлңЁлҰ°лӢӨ.

в–¶ Single Block I/O мҷҖ Multi Block I/O

  • Sigle Block I/O : н•ңлІҲмқҳ I/O Callм—җ н•ҳлӮҳмқҳ лҚ°мқҙн„° лё”лЎқл§Ң мқҪм–ҙ л©”лӘЁлҰ¬м—җ м Ғмһ¬н•ҳлҠ” л°©мӢқ
